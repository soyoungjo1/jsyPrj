# 2026-02-12 대화 요약 — 스크롤·4면 박스·포트폴리오

## 1. 수정·구현한 내용

### 1.1 메인 아스키 이후 아래 컨텐츠가 안 보이던 문제
- **원인**: 아스키 섹션이 `position: sticky` + `z-index: 9999`로 고정되어, 스크롤해도 아래 컨텐츠가 뒤에 가려짐.
- **해결**: 아스키 섹션을 **높이 200vh(또는 계산값)인 wrapper**로 감싸서, 스크롤 끝까지 가면 sticky가 풀리고 다음 섹션이 보이도록 구조 조정. (현재는 `cont.vue`에서 별도 sticky 스크롤 구간으로 구현됨.)

### 1.2 AsciiArt import 에러
- **에러**: `import { asciiArt } from '...'` — Vue SFC는 default export만 제공.
- **수정**: `import AsciiArt from '~/views/main/asciiArt.vue'`, 템플릿에서 `<AsciiArt />` 사용.

### 1.3 포트폴리오 디자인 컴포넌트 (`views/main/cont.vue`)
- 라벨·헤드라인·설명 + 작업 목록(리스트) + **Recent Project 3D 패널** 블록 추가.
- 이후 **스크롤 구간 인터랙션**으로 확장: 라벨 분리 → 가운데 이미지 등장 → 4면 박스 회전 → 다음 영역.

### 1.4 4단계 스크롤 인터랙션 (요구사항 정리)
1. 아스키 끝난 뒤 다음 섹션 진입 시, **라벨 두 개가 붙어 있다가** 스크롤 시작하면 **영역은 고정**, **텍스트만 양옆으로 이동**.
2. 문구가 양 끝에 도달했을 때 **가운데 이미지(박스) 생성**.
3. 스크롤하면 **우측으로 이동** — 총 **네 면의 상자**로 생각.
4. 스크롤해서 **다시 처음 이미지로 돌아오면** 다음 영역으로 이동.

### 1.5 4면 상자 — “이미지만 바뀌는” 문제 → **회전**으로 변경
- **기존**: 스크롤 시 보여줄 면 쌍만 바꿔서 이미지/텍스트만 교체 (평면처럼 보임).
- **목표**: **박스 전체가 세로축 기준으로 회전**하는 느낌. 스크롤 진행률에 따라 **회전 각도가 연속적으로 변하는 3D 회전**.
- **구현 요약**:
  - 4면을 3D 공간에 `rotateY(-i*90deg) translateZ(r)` 로 배치 (앞·오른쪽·뒤·왼쪽).
  - **스크롤 진행률(0~1)** 을 **회전 각도(0°~360°)** 로 매핑: `boxAngleDeg = stripT * 360`.
  - `.box-3d-rotating`에 `transform: rotateY(boxAngleDeg)deg` 적용 → 박스 전체가 한 덩어리로 회전.

### 1.6 스크롤 구간 높이 — 400vh 고정 → **계산값**으로 변경
- **변경**: `height: 400vh` 제거, **JS에서 계산**해 적용.
- **구현**: `SCROLL_STAGE_MULTIPLIER = 4`, `scrollStageHeightPx = window.innerHeight * SCROLL_STAGE_MULTIPLIER`, `onResize`에서 갱신. 템플릿에서 `:style="{ height: scrollStageHeightPx + 'px' }"` 바인딩.

## 2. 화면 고정 — “언제부터 고정? 어떻게 구현?”

- **언제**: “Recent Project” 스크롤 구간(400vh 또는 계산 높이)에 **진입하는 순간**부터, 그 구간을 **벗어날 때까지** 고정.
- **구현**:  
  - **부모**: `.recent-scroll-stage` — 높이를 크게 잡아 “스크롤 통로” 역할 (400vh 또는 `scrollStageHeightPx`).  
  - **자식**: `.recent-sticky-viewport` — `position: sticky`, `top: 0`, `height: 100vh`.  
  → 스크롤하는 동안 이 100vh 블록만 화면에 붙어 있음. **처음부터 sticky 한 번으로 구현.**

## 3. 스크립트 동작 요약 (4면 박스·스크롤)

- **진행률 계산**: `onScroll`에서 `scrollStageRef`의 `getBoundingClientRect()`·`offsetHeight`로 섹션 안 스크롤 비율 `p` (0~1) 계산 → `updateFromProgress(p)`.
- **단계 분리**: `PHASE_LABEL_END`, `PHASE_CENTER_*`, `PHASE_STRIP_*` 로 라벨 분리 / 가운데 등장 / 박스 회전 구간 구분.
- **회전 각도**: `stripT` (박스 구간 내 0~1) → `boxAngleDeg = stripT * 360`.
- **4면 배치**: `faceStyle(i)` → `rotateY(-i*90deg) translateZ(boxFaceRadius)`. 4면은 고정, 래퍼만 `rotateY(boxAngleDeg)` 로 회전.

## 4. 기타 질문·답변

- **ASCII 아트 필터 이미지**: 인물 사진 위에 ASCII 문자로 명암을 만든 **필터/효과**로 이해됨. 프로젝트의 AsciiEffect(3D 로고)와는 다르게, 2D 이미지를 ASCII로 변환하는 쪽.
- **BPCO 사이트 인터랙션**: 직접 실행은 불가. 구조·문구로 보아 로딩 퍼센트, 스크롤 유도, 스크롤 기반 섹션 전환 등이 있을 가능성 설명. Next.js 기반 추정.
- **인터랙션 설명만으로 구현 가능 여부**: 트리거(스크롤/호버/클릭), 동작(페이드/슬라이드/회전), 대상, 순서만 설명해 주면 구현 가능하다고 안내.
- **이 정도 스크롤 구현력**: 보통 **미들(3~5년)** 이면 요구사항만으로 설계·구현 가능. 주니어는 참고하면 따라 할 수준.
- **일주일 안에 마스터**: “완전 마스터”보다는 **일주일이면 개념 이해 + 지금 코드 따라 하기 + 비슷한 걸 한 번 직접 설계해 보기**까지가 현실적. 진짜 마스터는 여러 번 적용해 보며 쌓이는 단계. 일주일 학습 순서(스크롤 progress·sticky → 단계 나누기 → CSS 3D → 코드 정리 → 다른 시나리오로 연습) 제안함.
